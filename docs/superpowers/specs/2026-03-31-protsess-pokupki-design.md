## Layout страницы

Страница состоит из двух колонок на десктопе (≥1280px):

- **Основной контент** — `flex-1`, `max-w-3xl`, последовательные секции
- **Правый сайдбар** — фиксированная ширина ~160px, sticky, видим только на десктопе (`hidden xl:block`)

На мобильном сайдбар скрыт полностью, пользователь навигирует скроллом.

Общий контейнер: `max-w-[1200px] mx-auto px-4`.

---

## Правый сайдбар — навигация

- Sticky (`position: sticky; top: 96px`)
- Без рамок, фонов, теней — только текст
- Заголовок «На странице» — мелкий, `text-xs uppercase tracking-widest text-muted/40`
- Ссылки: `text-xs text-muted/50`, активная — `text-primary font-semibold border-l-[1.5px] border-primary pl-2`
- Неактивные — `border-l-[1.5px] border-transparent pl-2`
- Активная секция определяется через `IntersectionObserver` (`threshold: 0.3`)

## Секции страницы

Каждая секция получает уникальный `id` для якорной навигации и отслеживания IntersectionObserver.

### 1. Как мы работаем (`id="kak-my-rabotaem"`)

Вводный блок о философии компании. Текст + CTA-блок.

**Содержание:**
- Заголовок: «Как мы работаем»
- Текст: про честный подход, не «лишь бы купить», рекомендации знакомым
- CTA: «Подобрать автомобиль» → `/contacts`, «Получить честную консультацию» → Telegram

---

### 2. Процесс покупки (`id="protsess"`)

8 шагов в виде вертикального timeline (паттерн из `/dostavka` — пронумерованные круги с линией-коннектором).

**Шаги:**
1. Заявка и консультация
2. Подбор автомобиля (Copart, IAAI, Manheim; CARFAX; анализ повреждений)
3. Согласование и расчёт (финальные цифры до покупки)
4. Участие в торгах (онлайн, в офисе, или доверить нам; что если не выиграли)
5. Оплата и оформление (инвойс, SWIFT-перевод, аукционные сборы)
6. Доставка из США (по США → порт → море → Беларусь)
7. Таможенное оформление
8. Получение автомобиля (Гомель или другой город)

У каждого шага: номер-круг, заголовок, описание, при необходимости — вложенный список.

---

### 3. Оплата и договор (`id="oplata"`)

4 этапа оплаты + блок о договоре.

**Содержание:**
- Преамбула: работаем по договору, всё прозрачно
- 4 этапа (карточки или нумерованный список):
  1. Договор и предоплата за услуги
  2. Оплата автомобиля (лот + аукционный сбор + почтовые услуги)
  3. Оплата логистики (доставка по США + море + доставка в РБ)
  4. Таможенные платежи при оформлении в РБ
- CTA: «Не обещаем самые дешёвые авто — работаем так, чтобы вы рекомендовали нас» + кнопка

---

### 4. Доставка и растаможка (`id="dostavka"`)

5 этапов логистики + сроки + CTA.

**Содержание:**
- Вступление + общий срок (в среднем 6–10 недель)
- 5 этапов (timeline):
  1. Доставка по США до порта (3–10 дней)
  2. Подготовка и погрузка в контейнер
  3. Морская доставка (4–6 недель)
  4. Доставка в Беларусь
  5. Передача на таможню
- Блок о стоимости доставки (зависит от штата, типа авто, маршрута)
- Блок о трекинге («вы всегда знаете где авто»)
- CTA: «Рассчитать стоимость» → `/info/kalkulyator-rashod`, «Получить консультацию» → Telegram

---

### 5. Получение авто в РБ (`id="poluchenie"`)

Краткий блок.

**Содержание:**
- Куда приезжает: СВХ в Гомеле (или Минске)
- Что происходит: выгрузка → таможня → передача клиенту
- «Вы не остаётесь один» — сопровождение до момента получения
- CTA: «Рассчитать стоимость под ключ» → `/info/kalkulyator-rashod`, «Получить консультацию» → Telegram

---

### 6. Полезная информация (`id="info"`)

Блок «что важно знать перед покупкой». Структура: несколько тематических карточек или accordion.

**Реализация:** accordion (Accordion компонент уже есть в `src/components/ui/Accordion.tsx`)

**Темы:**
- С чего начинается покупка (бюджет, цель)
- Что важно учитывать (возраст авто до 5 лет, минимальный бюджет от $13 000)
- Из чего складывается стоимость (полный список составляющих)
- Как строится маршрут (до 1.5л → Клайпеда, от 1.5л → Поти) ⚠️ на странице /dostavka указано «до 3.0л» — уточнить у клиента
- 140 указ — льготная растаможка (инвалидность 1–2 гр., многодетные; 50% скидка)
- Риски при самостоятельной покупке (типы тайтлов, скрытые повреждения и т.д.)
- CTA: «Получить консультацию» + «Рассчитать стоимость»

## IntersectionObserver — механика

```tsx
'use client'
import { useEffect, useRef, useState } from 'react'

const sections = [
  { id: 'kak-my-rabotaem', label: 'Как мы работаем' },
  { id: 'protsess', label: 'Процесс покупки' },
  { id: 'oplata', label: 'Оплата и договор' },
  { id: 'dostavka', label: 'Доставка и растаможка' },
  { id: 'poluchenie', label: 'Получение авто в РБ' },
  { id: 'info', label: 'Полезная информация' },
]

// useEffect создаёт IntersectionObserver с threshold: 0.3
// При пересечении — setActiveSection(entry.target.id)
// Сайдбар читает activeSection и применяет active-стили
```
